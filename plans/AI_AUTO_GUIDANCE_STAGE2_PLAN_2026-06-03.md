# AI_AUTO 전역지침 2단계 — 패치비용 절감을 위한 정리 (2026-06-03)

백로그 후보(`ST-P1-32` 제안). `plans/AI_AUTO_GUIDANCE_CLEANUP_PLAN_2026-05-29.md`의
보류된 "Guidance Stage 2" 레인을 잇는다. 사용자 요청("전역지침 분리(패치비용 절감)")의
실제 목표는 **다운스트림 프로젝트의 템플릿 패치 비용 절감**이다.

본 계획은 측정 데이터와 AI 리뷰(Gemini + Codex, principal=claude) 1차 검토를 반영한
수정본이다. 1차 검토에서 드러난 사실:
- 완료팩 공용 스켈레톤(이전 슬라이스 B)은 **실익 없음**: 7개 팩 본문 360줄 중 2회+
  반복되는 줄이 12종뿐(대부분 코드펜스/한 문장). 스켈레톤화 시 절감≈0, 비용/위험만 발생
  → **B 폐기**.
- `AGENTS.md`를 참조로 외부화하면 시스템 프롬프트에서 규칙 본문이 사라짐(Gemini) →
  A는 "참조화"가 아니라 "문장 간결화"로.

## 1. 문제와 측정

다운스트림은 `AI_AUTO 최신 패치 적용해줘`로 템플릿을 갱신한다. **템플릿 소유** 지침이
클수록 패치 diff·하이브리드 병합·리뷰 비용이 커진다. 측정(2026-06-03):
- 템플릿 소유 대형 지침: `docs/AUTOMATION_OPERATING_POLICY.md` **650줄(루트↔템플릿
  byte-identical, verify 강제 동기)**, `docs/WORKFLOW.md` **340/템플릿 370줄**.
- `docs/MULTI_AI_COLLABORATION.md`(457)는 **템플릿 미러 없음(본진 전용)** → 다운스트림
  패치비용과 무관(본진 doc-budget에만 영향).
- `AGENTS.md`는 **160줄**로 150 예산 초과(본진 전용; 매 턴 시스템 프롬프트 주입).
- 루트↔템플릿 미러 중복은 의도된 배포본 → 제거 대상 아님.

결론: 다운스트림 패치비용의 실제 레버는 **템플릿 소유 대형 문서(운영정책 650 + 워크플로
340/370)** 이며, 완료팩·AGENTS는 아니다.

## 2. 목표

안전·검증·승인·완료·도메인 계약을 **한 글자도 약화시키지 않고**, 의도된 미러를 유지하면서:
(1) 템플릿 소유 대형 문서의 분량을 줄여 다운스트림 패치 표면을 축소하고,
(2) 본진 `AGENTS.md`를 예산 이하로 되돌린다.

## 3. 접근

### 슬라이스 A — 루트 AGENTS.md 간결화(본진 전용, 저위험)
규칙을 외부 참조로 빼지 않고 **문장만 간결화**해 150줄 이하로. 핵심 가드레일(verify/
review-gate 강제, 승인 없는 커밋·푸시 금지, 한글 명령 키워드, Completion Report Format,
Ralph 완료규율)은 **본문에 명시 유지**. 루트 전용 → 템플릿 버전범프 없음.
- 가드레일 보존 점검(Gemini): 압축 전 가드레일 규칙 목록을 작성해 압축 후 1:1 대조.
  과도한 단순화로 시스템 프롬프트의 규칙 준수력이 약화되지 않도록, 금지/필수 규칙의
  명령형 문장은 유지한다. 가능하면 압축 전후로 "승인 없이 커밋해줘" 류 레드팀 1회 비교.
- 중지조건: <=150줄, `doc-budget.sh` 경고 해소, `verify.sh`의 AGENTS 문자열 검사 통과.

### 슬라이스 C — 템플릿 소유 대형 문서 패치표면 축소(템플릿 소유, 고위험, 측정형 파일럿)
원칙: **규칙은 그대로, 표현만 줄인다.** 의미·규칙·게이트를 바꾸지 않는 **순수 기계적 압축**
(장황·중복 서술 정리, 동어반복 제거)만 수행. 외부 참조로 규칙을 빼지 않는다.
- 1차 대상: `docs/AUTOMATION_OPERATING_POLICY.md`(가장 큼, byte-identical 동기).
- **측정형 파일럿**: 먼저 한두 섹션만 압축해 절감률·의미보존을 확인하고 AI 리뷰로 검증.
  절감이 위험 대비 충분할 때만 나머지 섹션·`WORKFLOW.md`로 확대(후속 슬라이스 분리 가능).
- 템플릿 소유: 루트/템플릿 쌍 byte-identical 동기, `AI_AUTO_TEMPLATE_VERSION` 범프,
  `PATCH_NOTES.md` 상단 항목 추가.
- 안전장치: 각 규칙 문장은 의미 보존(삭제 아님). 압축 전후 규칙 목록 대조.
  - **문자열 참조 사전 스캔(Gemini)**: `verify.sh`뿐 아니라 `scripts/`, `tests/`,
    `.github/`(CI) 전반에서 대상 문서의 특정 문장/단어를 grep으로 검증하는 곳이 있는지
    먼저 조사하고, 압축이 그 매칭을 깨지 않게 한다(혹은 해당 검사도 함께 갱신).
  - **다운스트림 전환 비용 고지(Gemini)**: 대형 압축은 일시적 대용량 diff라, 해당 문서를
    하이브리드로 커스텀한 다운스트림은 다음 패치 때 병합 갈등이 커져 단기 패치비용이
    오를 수 있다. AI_AUTO 패치 워크플로가 하이브리드 파일을 review-merge로 표면화하므로
    조용한 손실은 아니지만, 이 1회성 전환 비용을 패치노트에 명시하고 큰 압축은 한 번에
    몰아 단발 전환으로 끝낸다(찔끔 반복 금지).

### 보류
- `MULTI_AI_COLLABORATION.md`(본진 전용)의 로드맵/이력 슬림화는 **다운스트림 패치비용과
  무관**하므로 이번 목표에서 제외(원하면 본진 doc-budget용 별건으로).
- 완료팩 스켈레톤(구 B): 측정상 실익 없어 폐기.

## 4. 비목표(Non-Goals)
- 안전/승인/검증/완료/도메인 계약·게이트·규칙의 변경·삭제 금지(압축은 표현만).
- 규칙을 다른 문서 참조로 외부화 금지(시스템 프롬프트/온디맨드 컨텍스트에서 규칙 유지).
- 의도된 루트↔템플릿 미러 제거 금지.
- 경고만 근거로(검증 없이) 문서 수정 금지.

## 5. 검증(엔드투엔드)
1. `doc-budget.sh`: `AGENTS.md` <=150, 템플릿 지침 총량 감소, 신규 실패 없음.
2. `verify.sh` 그린 — 특히 AUTOMATION_OPERATING_POLICY byte-identical 동기, 운영정책
   문자열 검사, 템플릿 버전/패치노트 검사, AGENTS 문자열 검사.
3. `AI_AUTO_PRINCIPAL=claude ./scripts/review-gate.sh` (run-ai-reviews + summarize를
   감싸는 정식 게이트) → Gemini + Codex 만장일치(proceed). 압축은 의미 변경 위험이 크므로
   리뷰에서 규칙 보존을 집중 확인.
4. 충족 시 **커밋 후보 제시**. 커밋·푸시는 사용자 명시 승인 후에만(승인 없는 커밋·푸시
   금지 규칙). 승인 후 백로그 `ST-P1-32` complete_contract 승격.
