# AI_AUTO later_gated 항목 통합 시퀀싱 계획 (2026-06-03)

사용자 요청: 백로그의 later_gated 3건(ST-P1-20/21/33)의 **우선순위·의존성·발동조건**을
하나로 정리하는 상위 계획. 이 문서는 구현 계획이 아니라 **거버넌스/순서 계획**이다.
어떤 항목도 본 문서만으로 active가 되지 않으며, 각자의 트리거 + 사용자 승인 후 별도
구현 계획으로 진입한다. AI 리뷰(Gemini + Codex, principal=claude)로 타당성 검증한다.

## 항목별 평가

### ST-P1-21 — MicroWork (가치 높음, 의존성 없음)
- 가치: 이번 세션처럼 "마이크로 단위 실행 + 스코프 규율 + 완료조건 드리프트 방지(ST-P1-28)"를
  **프롬프트 규칙이 아니라 검증 가능한 도구**로 형식화. 현재 작업 방식과 직접 정합.
- 범위(승인된 한계): 부작용 없는 `tools/micro-work` JSON 검증기 + 얇은 `scripts/micro-check.sh`
  래퍼 + 타깃 pytest + `verify.sh` 스모크 + `collect-review-context.sh` 리포트 전용 감사
  (스코프 드리프트/증거 적정성/비목표 누수/smallest-useful-wedge). **런타임·큐·스케줄러·UI·
  자동실행·완료권한 신설 금지.**
- 의존성: 없음. 사용자가 곧바로 활성화 가능.
- 참고: 사전 탐색 `plans/AI_AUTO_SMALL_TOOL_MICRO_EXECUTION_2026-05-29.md`.

### ST-P1-20 — 도메인팩 회고 (가치 중, 트리거 미충족)
- 가치: AI_AUTO의 근본 루프(프로젝트 경험 → 본진 환원)를 도메인팩에 적용. 재사용 가능한
  것만 `templates/domain-packs/<name>/`로 승격.
- 범위: 프로젝트 종료 시 1회성 시퀀스(피드백 수집 → 재사용 vs 프로젝트 고유 분류 → 일반화분만
  승격). **상시 도메인팩 피드백 루프 신설 금지.**
- 의존성: **종료된 프로젝트 1개 이상**(증거 필요). 현재 jw_dev/zurini 진행 중 → 트리거 미충족.
  → 지금 구현 부적합, 첫 프로젝트 종료 시 활성화.

### ST-P1-33 — 정책문서 압축 (가치 낮음, 상위 결정에 종속)
- 2026-06-03 측정: 운영정책은 밀집 규칙이라 압축 여지 ~3-5%, verify가 본문 다수 문자열 grep,
  안전 위험 큼 → 효과 불충분.
- **상위 의존성**: 패치비용 문제 자체가 "공용지침 복사 배포" 모델의 부산물이다. 보류 중인
  "중앙참조 vs 복사" 아키텍처 결정에서 **참조 모델을 택하면 패치비용≈0**이 되어 압축 동기가
  사라진다. 따라서 ST-P1-33은 그 아키텍처 결정 **전에는 추진하지 않는다**(중복 노력 위험).
- → 최하위. 아키텍처 결정 후 재평가, 그때도 압력이 실재할 때만.
