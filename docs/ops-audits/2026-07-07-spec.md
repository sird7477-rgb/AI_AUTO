## IP-1 [P0] Verification Contract — 런타임 오라클을 승격 필수조건으로

### 문제
정적 green(`verify-project.sh` = XML+py_compile)이 push-ready로 취급됨 → 무한재귀 커밋 `5e7da49`가
origin 도달 + odoo.sh KILLED. 실오라클(docker harness `validate-full.sh`)은 미배선·docker-down시 silent-skip.

### 요구 (AC)
- **AC1-1 (proof 존재·신선·커밋바인딩)**: push 승격은 `PROOF{commit=<HEAD sha>, tree=<tree sha>,
  harness_version=<ver>, docker=available, result=PASS, ts, hmac}` 를 요구. HMAC은 out-of-tree 키로
  서명(`ai-principal-runtime.sh`/`review-gate-binding.sh` 기존 패턴 재사용). **proof.commit ≠ 현재 HEAD면
  거부**(stale/다른-HEAD 재사용 차단).
- **AC1-2 (docker-down = hard stop, not degraded)**: harness가 docker 부재로 못 돌면 proof는 PASS가 될 수
  없고 승격은 **차단**. "일시 degraded"로 통과 금지. (본 세션 #1 harness-preflight가 docker-down을 명확히
  신호 — 그 신호를 proof=FAIL로 승격.)
- **AC1-3 (미배선 = hard stop)**: harness/helper 스크립트 부재 → 정적만 돌고 통과가 아니라, **NOT-VALIDATED
  proof=FAIL**로 승격 차단(기존 커밋 `0bd963c` 스캐너 LOUD NOT-VALIDATED 패턴을 오라클 부재로 확장).
- **AC1-4 (wrong-worktree 차단)**: proof는 실행 워크트리 경로/토플레벨을 기록, 승격하는 트리와 불일치면 거부.
- **AC1-5 (오라클 비공허)**: proof=PASS는 harness가 실제로 `-u`/테스트를 돌려 non-empty 로그를 냈음을 증거로
  요구(VMEXIT/결과라인). 로그 없음·빈 로그 = PASS 아님(green-by-absence 금지).

### 안티-위조 회귀테스트 (필수, 게임 레드가 재공격)
- R: PASS proof 생성 → HEAD를 1커밋 전진 → 승격 시도 = **거부**(AC1-1).
- R: docker stop 상태서 승격 = **거부**(AC1-2).
- R: harness 스크립트 rename/삭제 후 승격 = **거부**(AC1-3).
- R: proofA를 다른 워크트리에 복사 후 승격 = **거부**(AC1-4).
- R: harness 로그를 빈 파일로 위조 = **거부**(AC1-5).

## IP-2 [P1] Gate/harness presence fail-closed across worktrees

### 문제
`review-gate.sh` 등 게이트 스크립트가 일부 워크트리엔 없음 → 커밋이 post-commit **경고**만 남기고 통과.
`--no-verify` + 원시 git 플러밍(write-tree/commit-tree)이 상시 우회. session-lock 부재로 공유 워크트리 오염.

### 요구 (AC)
- **AC2-1 (bootstrap-detect LOUD)**: `automation-doctor.sh --bootstrap`(신규 모드)가 프로젝트에 필수 게이트/
  harness/lock 배선 부재를 탐지 → **비-0 exit + 명시 목록**. pre-push가 이를 호출, 미배선이면 push 차단
  (경고 아님).
- **AC2-2 (post-commit warning → promotion block)**: "may have bypassed review-gate" 경고 조건은
  **push 시점에 hard-block**으로 승격(커밋은 로컬 자유, 그러나 미검증 커밋의 push는 차단).
- **AC2-3 (--no-verify/plumbing 탐지)**: pre-push가 최신 커밋들의 바인딩 proof 부재를 탐지하면 차단 —
  `--no-verify`로 훅을 건너뛰어도 서버는 없지만, pre-push 자체는 우회 흔적(바인딩 없는 HEAD)을 승격 차단.
  (플러밍 커밋도 바인딩 없으면 동일 차단.)
- **AC2-4 (session-lock 배포)**: `session-lock.sh`를 도메인팩 배선 필수 항목으로, 부재시 AC2-1이 탐지.

### 회귀테스트
- R: 게이트 스크립트 없는 워크트리서 push = **차단**(AC2-1).
- R: 바인딩 proof 없는 HEAD push = **차단**(AC2-2/3).
- R: `git commit-tree` 수제 커밋 push = **차단**(AC2-3).

## IP-3 [P1] 독립 cross-family 리뷰 복원

### 문제
Gemini(agy)가 `prompt_bytes=102029 > prompt_arg_max_bytes=100000`에서 `large_prompt_requires_prompt_file`로
retry_exhausted → **21일간 persistent disabled**, `next_action=user_reset_required`. Claude 리뷰어도 disabled.
매 게이트가 codex 자기리뷰로 degrade. (`ai-runtime-adapter.sh:296/327`, `review-gate.sh:641/656`.)

### 요구 (AC)
- **AC3-1 (대용량 → prompt-file 폴백)**: `run_readonly_agy`가 `prompt_bytes > arg_max`일 때 에러 대신
  agy `--prompt-file` 경로로 전달(agy CLI가 지원). arg-max는 arg 전달 한계일 뿐 파일경로엔 무관.
- **AC3-2 (그래도 실패시 청킹 + synthesis)**: prompt-file로도 한계 초과면 diff를 청크 분할 리뷰 후 **최종
  synthesis 패스**로 병합, 결과에 `review_mode=chunked, chunks=N, truncated=<bool>` 메타 부착(맥락손실 고지).
- **AC3-3 (size-fail은 transient)**: 프롬프트 크기/prompt-file 관련 실패는 **transient**로 분류(자동 cooldown
  재프로브), persistent(user_reset_required) 금지 — 크기는 다음 런에서 다른 diff면 회복됨. `review-gate.sh:656`
  transient auto-recovery 경로로 라우팅.
- **AC3-4 (api_env=missing은 정직 degrade)**: 자격증명 부재는 tool이 못 고침 → proceed_degraded 유지하되
  **degraded 사유를 gate 요약에 lineage로 표기**(single-family 표기), 그리고 **stale disable 21d 경고를
  push 승격 시 non-fatal이 아니라 P0 리뷰가 필요한 변경(예: 스키마/삭제/상태전이)에는 block**으로 승격.

### 회귀테스트
- R: 102029B 프롬프트 → adapter가 prompt-file 경로로 non-error 실행(AC3-1) [agy 부재환경은 mock].
- R: size-fail 마커가 transient로 분류되어 cooldown 후 재프로브(AC3-3).
- R: chunked 리뷰 결과에 synthesis + meta 존재(AC3-2).
