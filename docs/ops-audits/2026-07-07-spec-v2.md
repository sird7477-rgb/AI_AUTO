# SPEC v2 — 적대리뷰 반영 교정본 (2026-07-07)

v1(`2026-07-07-spec.md`)을 red-team(Sonnet) + cross-family(Codex)가 공격. 치명결함 3종으로 v1은
**블루팀 인계 부적격** 판정. 본 v2가 교정본이자 #4 블루팀 정본 브리프. (원문·리뷰 로그 보존.)

## 교정 원칙 (리뷰 D1/D2가 강제한 재프레이밍)

- **집행모델 정직화 (D1)**: git 훅은 clone/`worktree add`/push로 전파되지 않는다 — 이게 A1의 뿌리다.
  따라서 훅은 **dev-time 편의**이지 authoritative 게이트가 **아니다**. 정본 게이트는 **push 대상에서
  독립적으로 도는 out-of-band auditor**여야 한다. 헤드라인을 실제로 잡은 것도 odoo.sh 원격빌드였다.
- **위조내성 주장 하향 (D2)**: same-UID 셸이 서명 헬퍼를 직접 불러 harness 미실행 상태로 유효 HMAC proof를
  주조할 수 있다(위협모델=untrusted project지 operating shell 아님). 그러므로 IP-1 proof는
  **"위조불가"가 아니라 "비용상승 + 감사추적 + defense-in-depth"**로 정직 표기. hard-close 주장 금지.

## 재구성된 페이즈 (D3/D4/D5 반영)

**PHASE-1 (co-required, "헤드라인 fixed"의 정의):** IP-1 + IP-2 + Lore-hook FP를 **함께** 착수.
IP-1 단독은 게이트 없는 워크트리(헤드라인 발생지)를 못 잡으므로 IP-2와 분리 불가. Lore-hook FP는 우회의
실증 트리거이므로 IP-2가 게이트를 강화하기 전에 동반 수정(안 그러면 plumbing 우회 압력 증가).

## IP-1' Verification Contract (교정)

- **AC1-1' (proof 바인딩, 정직표기)**: v1 AC1-1~AC1-5 유지하되 헤더에 "same-UID 위조는 이 계약의
  위협모델 밖(비용상승·감사추적 목적)"을 명시.
- **AC1-6 (신규, D2 필수테스트)**: 서명 헬퍼로 harness 미호출 상태의 proof 주조 시도 → 시스템이 이를
  **거부 못 함을 테스트로 문서화**(negative-knowledge fixture). 대신 **out-of-band auditor가 harness
  실행로그 부재를 사후 탐지**함을 positive-control로 요구. (탐지≠예방을 정직하게 분리.)
- **AC1-7 (신규, D1 out-of-band auditor)**: 훅 설치 여부와 무관하게 도는 auditor(스케줄/독립 스캔)가 라이브
  워크트리들의 (HEAD, 최신 proof) 정합을 검사 → 미검증 HEAD·harness로그 부재를 리포트. 이것이 정본 백스톱.
- **AC1-8 (D8 scope 확장)**: `mods` 비었어도(custom-addons 밖 변경) skip 금지 — 임의 diff엔 harness를
  돌리되 `-u` 대상만 mods로 좁힘. regex-회피 경로 회귀테스트 추가.
- **AC1-2' (D7 ack-bypass 정직화)**: docker-down hard-stop과 기존 `odoo_unvalidated_ack_or_block`의
  충돌 해소 — ack-bypass는 `AI_AUTO_PRINCIPAL_LAUNCHER=1` env(셸 자가export 가능)만으론 불충분.
  **셸이 자가생산 못 하는 신호(interactive TTY 확인)로 상향**하거나 ack-bypass를 P0변경(스키마/삭제/상태전이)엔
  불허. 미해소면 SPEC이 "override 존재"를 명시.
- **AC1-9 (D10 TOCTOU + lock)**: proof는 `session-lock.sh` 토큰에 바인딩. proof생성~push 사이 트리
  변이(다른 세션) 시 무효. 워크트리경로 문자열(AC1-4)은 이걸로 대체.

## IP-2' Gate presence fail-closed (교정)

- **AC2-1' (D9 열거)**: `automation-doctor --bootstrap`가 required로 승격하는 파일/락 **정확한 목록 명시** +
  기존 `--project` warn-only 시맨틱 불변 회귀테스트. (early-stage 프로젝트 오차단 방지.)
- **AC2-5 (D5 hard-block 조건부 활성)**: IP-2 push hard-block은 **IP-3의 non-codex 리뷰어가 1명 이상
  라이브가 된 뒤 활성**. 그 전엔 "codex 자기승인"으로 축소되므로 block은 audit-log/경고에 머문다(정직 표기).
- **A5' Lore-hook FP (D4 승격)**: 규정준수 메시지 8회 거부한 pre-commit Lore-format 훅 false-positive를
  PHASE-1에 수정(ai-lab 소유 훅 한정). 회귀: 정준수 메시지가 1패스 통과.

## IP-3' 리뷰 복원 (교정)

- AC3-1~AC3-4 유지 + **AC3-5 (D6 chronic counter)**: size-fail을 transient로 돌리되 동일 `reason`의
  재-disable가 반복되면 `disabled_at` 리셋으로 stale경보가 영원히 안 뜨는 문제 → **freshness와 독립한
  chronic-redisable 카운터**가 임계 넘으면 P0경보(AC3-4) 유지. `--prompt-file` 지원 판정이 `--help` grep
  false-negative일 수 있으니 실행실패도 chronic으로 집계.
- **AC3-2' (D11)**: chunked synthesis 테스트는 presence가 아니라 **청크경계를 걸치는 이슈를 synthesis가
  실제 포착**하는지로 강화.

## IP-4' 잔여 (D12)

- D2 venv gap: "shell-only" = "diff가 pytest-collected 루트 아래 경로를 하나도 안 건드림"으로 **정의**
  후 구현(확장자 기준 금지).
